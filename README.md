# jepa-transformer-edge-profiling

A benchmarking and performance analysis tool designed to profile Transformer layer across different hardware architectures (CPU vs. GPU). This project evaluates computational latencies of self-supervised lightweight model based on Vision Transformer (dinov2 ViT-small).

---

## Notebook Structure (Google Colab)

The codebase is engineered and divided into 3 independent logical blocks to optimize runtime switching (CPU/GPU) without any data loss

### Setup and dataset management
* Connects Google Drive and verifies the presence of a real image dataset in the `dataset_test/images` directory.
* If the folder is empty during the first execution, the script automatically downloads a configurable number of real test images via public APIs, ensuring consistency and repeatability of the benchmark across different hardware environments.

### Executive profiling
* Applies PyTorch's `register_forward_pre_hook` and `register_forward_hook` directly onto the target submodules of the Transformer.
* **On CPU:** Utilizes high-precision temporal registers via `time.perf_counter()`.
* **On GPU (CUDA):** Utilizes `torch.cuda.Event(enable_timing=True)` combined with asynchronous synchronization (`torch.cuda.synchronize()`) to completely isolate CPU overhead and capture pure hardware execution time.
* At the end of the run, it automatically exports a structured `.csv` report directly to your Google Drive folder.

### Comparative chart generation
* Reads the CSV files generated by the respective hardware pipelines (CPU and GPU) and performs an automatic data merge.
* Generates a side-by-side bar chart using a **dual Y-axis layout**.

---

## Dependencies

The main dependencies utilized within the Colab notebook are:
* **torch & torchvision** – ViT model loading and tensor manipulation.
* **pillow & requests** – Image I/O operations and automated image downloads.
* **pandas** – Data aggregation and structured layer metrics handling.
* **matplotlib & numpy** – Data visualization and plotting.
